Site Build Management
Add Client
  • Go to Flight School, hover on Clients on the top left corner of the page, and then click Add Client.
  • Fill out the page based on the information on FS
    • Client Name (Internal): Same as the school name on FS + , Space + State Abbreviation (e.g. Hogwarts School, OR) or * + the two letter Country Abbreviation" (Hogwarts School, *GB)
    • Display Name (Site): Same as the school name that will display on the Atlas site (e.g. Hogwarts School)
    • Exemplar License: Check it only for clients (not for evaluative prototypes, internal sites, or other cases)
    • Display Public Site in Communities: Do NOT check it. This is for making the public site link available to all Atlas users.
    • State/Country (Internal) / Location: Based on the information on FS
    • Version: 8.2
    • Production URL: Based on the school website url and NO special charactors other than - (but try NOT to use more than one), letters and digits only!
    • Rubicon lead: Based on the information on Flight School
    • Leave everything else blank, save it, and then you will see the Flight School option. Attach FS to this page, and then save the page again.
Create URL
  • Copy the highlighted part of the URL: https://hogwarts.rubiconatlas.org
  • Connect to Web1 (remote Desktop) and log in.
  • Go to Window Start and RIGHT CLICK the Command Prompt and CLICK Run as Administrator.
  • Create URL:
    • New URLs: newsiteSpaceURL, and then Enter
    • Delete URLs: deletesiteSpaceURL, and then Enter
    • Rename URLs: renamesiteSpaceold URLSpacenewURL, and then Enter
    • Public Site URLs: addpublicSpaceroot URL, and then Enter
    • New .NET URLs (e.g. for Test Drives): newsite.NET.batSpaceURL, and then Enter
Select Database
  • Basic Consideration: Content in SET, Demo Data or Module Curriculum, Categories and Templates, Schools under same organizations/associations, etc
  • Commonly Used Shell Systems:
    • UbD Basic - for most of the UbD sites
    • NYC Reboot - for NYC sites and NYC quick site builds
    • OS/MAISA - for schools need OS/MAISA curriculum modules
    • Engage NY - for schools need EngageNY curriculum modules
    • AU/Roos - for schools in Australia
    • IB Resources - for IB schools
    • JL Reboot - for Joseph's schools
  • Process:
    • Go to the "little man" page of the shell system, and then click the "Restore" button.
    • Give a name for the database you are creating (see name convention below). DO NOT use charactors other than letters and digits or you are fired!
    • Database Name Convention: Atlas+URL (e.g. the database name for hogwarts.rubiconatlas.org is AtlasHogwarts (case sensitive)).
    • Copy the Sever Name and Database Name and pasted them into the new site's School Constants > Database/Development > SQL_DATA_SOURCE and SQL_INITIAL_CATALOG respectively, and then save.
